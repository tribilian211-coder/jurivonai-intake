  ⚠️  IMPORTANT: GitHub's web upload DOES NOT properly upload
  folders like src/, prisma/, and public/. If you drag-dropped
  files into GitHub's website, your repo is MISSING the actual
  application code — and the deploy will fail.

  Use ONE of the two methods below instead. Both are free and
  take ~5 minutes.

═══════════════════════════════════════════════════════════════════
  METHOD 1 — GitHub Desktop (EASIEST, recommended for non-coders)
═══════════════════════════════════════════════════════════════════

  1. Download GitHub Desktop from: desktop.github.com (free)
  2. Install it, sign in with your GitHub account
  3. Click "File" → "New Repository"
  4. Name: jurivonai-intake
  5. Local path: click "Choose" → select THIS folder
     (the one containing START-HERE.txt)
  6. Click "Create repository"
  7. You'll see a list of all files on the left — verify you see
     src/, prisma/, and public/ folders listed
  8. In the "Summary" box at bottom, type: "Initial commit"
  9. Click "Commit to main"
 10. Click "Publish repository" (top right)
 11. Choose your GitHub account, keep the name "jurivonai-intake"
 12. Click "Publish repository"
 13. Done! All files AND folders are now on GitHub.

═══════════════════════════════════════════════════════════════════
  METHOD 2 — PowerShell script (no extra software beyond Git)
═══════════════════════════════════════════════════════════════════

  Pre-requisite: install Git from https://git-scm.com/download/win

  1. Create an EMPTY repo on GitHub:
     - Go to github.com → click + → New repository
     - Name: jurivonai-intake
     - DO NOT check any "Initialize" boxes (no README, no license)
     - Click "Create repository"
     - Copy the repo URL (looks like:
       https://github.com/your-username/jurivonai-intake.git)

  2. In THIS folder (where START-HERE.txt lives), find the file:
        deploy-to-github.ps1

  3. Right-click it → "Run with PowerShell"

  4. When prompted, paste your GitHub repo URL and press Enter

  5. The script will upload ALL files and folders (src/, prisma/,
     public/, etc.) to GitHub automatically.

  6. If a GitHub login window pops up, sign in to authorize Git.

═══════════════════════════════════════════════════════════════════
  AFTER YOUR CODE IS ON GITHUB — Deploy to Vercel
═══════════════════════════════════════════════════════════════════

  1. Create free database: neon.tech → sign up → copy the
     "Connection string" (starts with postgresql://...)

  2. Create the table: in Neon → SQL Editor → New query →
     paste the contents of prisma/migration.sql → click Run

  3. Deploy: vercel.com → sign up with GitHub → "Add New" →
     "Project" → import "jurivonai-intake" → expand "Environment
     Variables" → add:
        DATABASE_URL       = (your Neon connection string)
        ADMIN_PASSWORD     = (pick a strong password)
     → click Deploy → wait 2 minutes → click Visit

  4. Test: open your Vercel URL → type a test email → click
     "Begin interview" → answer a few questions

  5. Admin dashboard: add #admin to your URL
     (e.g. https://your-app.vercel.app/#admin)
     Enter the ADMIN_PASSWORD you set in step 3.
